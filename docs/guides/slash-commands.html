Slash Commands · 斜杠命令

Slash commands are typed into the REPL starting with /. The full registry is a compile-time static array SLASH_COMMAND_SPECS at crates/commands/src/lib.rs:73–300, parsed by SlashCommand::parse() (commands/lib.rs:376–462).

Categories

Defined at crates/commands/src/lib.rs:43–60.

Category Examples
Core /help, /model, /vim, /clear
Workspace /permissions, /add-dir
Session /session, /resume, /export, /compact
Git /branch, /worktree
Automation /plugin, custom hooks

Discovery

Method Action
/help List all commands grouped by category
Tab in REPL Cycle completion over known commands (input.rs:689–746)
docs/reference/slash-commands.md Full table with arguments

Argument styles

  • Positional — the common case: /model [model], /permissions [mode], /branch [action] [target], /worktree [action] [path] [branch], /session [action] [target], /plugin [action] [target].
  • Flag-style — only /clear --confirm accepts a flag (commands/lib.rs:425–426).

Aliases

Both /plugins and /marketplace map to /plugin. Other commands have no aliases.

See also

  • sessions.md for /session, /resume, /export, /compact.
  • docs/reference/slash-commands.md for the complete per-command argument list.

参数风格

  • 位置参数 — 常规场景：/model [model]、/permissions [mode]、/branch [action] [target]、/worktree [action] [path] [branch]、/session [action] [target]、/plugin [action] [target]。
  • flag 风格 — 仅 /clear --confirm 接受 flag（commands/lib.rs:425–426）。

别名

/plugins 与 /marketplace 都映射到 /plugin，其他命令无别名。
