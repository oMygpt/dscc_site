案例库

可复现的端到端场景。每个案例自成一体：PROMPT.md 是可直接灌给 DSCC 的 prompt，README.md 说明准备工作、预期工具调用顺序以及如何验收。

每个案例只演示一种能力、保持 prompt 简短，描述模型行为时用“通常会“而非“一定会“。

案例列表

# 目录 一句话主题 演示能力
01 01-code-review 只读模式复盘一个 Rust 文件，找出预埋的 bug grep_search + read_file + StructuredOutput、ReadOnly 权限
02 02-test-generation 为无测试的工具函数生成表驱动单测 read_file → write_file、WorkspaceWrite 权限
03 03-refactor-rename grep + edit 在 4 个文件里重命名函数 grep_search → edit_file（replace_all）、WorkspaceWrite
04 04-web-research 抓取在线 API 参考并汇总成表格 WebSearch + WebFetch、ReadOnly
05 05-data-analysis 对 25000 行 CSV 做统计并输出报告 bash + 文件读写、DangerFullAccess

如何阅读

  1. 打开对应的 README.md：背景、准备、运行命令、预期工具序列、验收方式。
  2. 打开 PROMPT.md，用 dscc prompt "$(cat PROMPT.md)" 或管道灌进去。
  3. 对照验收条目检查模型的实际输出。
