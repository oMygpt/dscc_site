Read src/utils.rs. Generate table-driven unit tests for parse_duration covering:

  • happy path ("10s", "2m", "1h")
  • edge cases (zero, whitespace, unit-only, very large numbers)
  • error cases (empty string, unknown unit, negative values, non-numeric prefix)

Write the tests to tests/utils_test.rs. Use #[test] functions and a single shared test table. Do not modify src/utils.rs.
