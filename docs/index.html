Document style · 文档风格

  • Facts first: every non-obvious claim cites a file path or a file:line reference into this repository so readers can verify.

  • No fluff: tables over prose; short imperative sentences.

  • Bilingual discipline: English section first, then ### 中文 with the same structure. Code blocks, file paths, and commands are never translated.

  • Verified commands: any shell command that has not been tested on a real machine is annotated with # [待验证]. Contributors drop the marker only after testing.

  • 事实优先：不显然的结论都要给出文件路径或 file:line 引用，读者可复核。

  • 少废话：能用表格就不用散文，句子用祈使式。

  • 双语规范：英文在前，### 中文 小节跟在后面，结构保持一致。代码块、路径、命令不翻译。

  • 命令验证：未在真实设备上验证过的 shell 命令用 # [待验证] 标注；贡献者验证通过后再去掉。
